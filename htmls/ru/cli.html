Интерфейс командной строки:
параметры и входные файлы могут быть в любом порядке

Внимание:
Командная строка не загружает GUI компоненты, за исключением параметра [-gui].

без параметров
..запуск с GUI.

параметры:
-ini <путь + файл.ini>
..чтобы использовать не стандартный файл.ini.
..все настройки для GUI (включая посткоманды) управляются непосредственно через файл .ini, кроме пользовательских переменных [-ini,-name,-cut,-chp,-id,-saveini,исходный файл].

-dvx1
..создать файл проекта .d2v при демультиплексировании

-dvx2
..создать файл проекта .d2v при демультиплексировании
..устанавливает параметры добавления заголовка RIFF Wave для АС3 файлов

-dvx3
..создать файл проекта .d2v при демультиплексировании
..устанавливает параметры добавления заголовка RIFF Wave для MPEG аудио файлов

-dvx4
..создать файл проекта .d2v при демультиплексировании
..устанавливает параметры добавления заголовка RIFF Wave для АС3 файлов
..устанавливает параметры добавления заголовка RIFF Wave для MPEG аудио файлов

-out <путь + имя каталога>
..использовать указанный каталог для вывода.

-name <имя файла>
..использовать указанное имя как базовое имя для всех выходных файлов.

-cut <путь + файл>
..используйте этот текстовый файл в качестве списка точек обрезки.

-chp <путь + файл>
..используйте этот текстовый файл как список точек глав.

-id <значения>
..используйте только эти (P)IDs, отделенные запятой ",".

-gui
..отображение GUI с использованием всех предложенных через командную строку параметров.

-log
..записать обычный файл журнала.

-saveini
..сохранить изменения, использованные командной строкой в активном .ini.

-split <xxx>
..разделение вывода на xxx МБ.

-demux, -tom2p, -topva, -tovdr, -tots, -filter
..типы действий.

Для создания специального файла .ini, запустите GUI, установите все необходимые вам параметры и сохраните где вам угодно и под любым именем (настройки производятся в - основное окно\Предустановки\Предустановки, там же и сохраняются в файл - основное окно\Предустановки\Предустановки\Файл\Сохранить как). Аналогично для создания файла точек обрезки и файла точек глав (настройки производятся в - основное окно\Управление сокращением, там же и сохраняются в файл - основное окно\Управление сокращением\Точки обрезки или Главы\кнопка с изображением дискеты).

Отображение Русского языка(кириллицы) в окне консоли Windows:

 Если в окне консоли вместо русских букв отображаются кракозябры (непонятный набор символов) - необходимо выполнить следующее :

 Изменить шрифт отображения в консоли:
  В окне консоли щелкнуть правой кнопкой мыши по верху рамки окна консоли и в появившемся меню выбрать "Свойства\Шрифт" - выберите шрифт "Lucida Console" или любой другой поддерживающий кириллицу(если имеется).
  (изменение шрифта требуется однократно (т.е. при следующих запусках консоль будет отображаться с этим шрифтом))

 Изменить кодовую страницу консоли:
  Начать работу в консоли с параметра "chcp 1251" (команду необходимо вводить при каждом открытии окна консоли (т.е. она действует только в том окне - в каком была запущена и до закрытия окна)).
  где
   chcp [xxx] - Установка активной кодовой страницы.
   xxx - номер кодовой страницы.
   chcp - без параметра выведет текущий номер кодовой страницы.
   866 - номер кодовой страницы в консоли Windows по умолчанию.
   1251 - номер кодовой страницы поддерживающей русские буквы.
  Пример:
   C:\Windows\system32>chcp 1251
   - нажатие клавиши "Enter"
   - отобразится
   C:\Windows\system32>chcp 1251
   Текущая кодовая страница: 1251

   C:\Windows\system32>
   - далее работаем как обычно
   C:\Windows\system32>chcp 1251
   Текущая кодовая страница: 1251

   C:\Windows\system32>java -jar "C:\Portable Programs\De(Multiplexer) \Project-X\ProjectX.jar" [параметры] <файл>
   - и т.д.

   или

   C:\Windows\system32>chcp 1251&java -jar "C:\Portable Programs \De(Multiplexer)\Project-X\ProjectX.jar" [параметры] <файл>
   где
    & - разделитель команд в командной строке (позволяет прописать несколько команд в одной строке).
    или
    && - команда, следующая за этим символом будет запущена, только если команда стоящая перед символом, завершилась успешно (также позволяет прописать несколько команд в одной строке).

  (само собой - можно использовать в (.bat) файлах)
